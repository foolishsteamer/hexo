1.scss是什么？在vue.cli中的安装使用步骤是？有哪几大特性?

scss是一种css预处理器(sass的最新版本),它可以让我们写一些高级的css,如规则嵌套、变量、混合、继承等等特性，让CSS的书写更加有趣与程式化。

vue.cli的安装步骤:

特性:

规则嵌套、变量、混合、继承等等特性，让CSS的书写更加有趣与程式化。

2.那么多框架,项目为什么选择用vue?有什么特点?

vue可以帮助我们构建复杂的spa页面,可以让前端页面像native app,拥有更好的用户体验.vue的对中文友好,社区活跃,第三方插件多.vue的核心特点是数据双向绑定和组件化.

3.请说下你对vue声明周期的理解?他的作用?

Vue 实例从创建到销毁的过程，就是生命周期。

作用是在vue实例创建和销毁时的几个阶段,调用对应的生命周期函数,可以在对应的时间里做一些操作,如计算数据、调用方法等.

它可以总共分为11个阶段：创建前/后, 载入前/后,更新前/后,keep-alive组件激活前/后,销毁前/销毁后,子孙组件报错时.

<keep-alive>是Vue的内置组件，能在组件切换过程中将状态保留在内存中，防止重复渲染DOM。

<keep-alive> 包裹动态组件时，会缓存不活动的组件实例，而不是销毁它们。

4.第一次页面加载会触发哪几个钩子

beforeCreate created beforeMount mounted

5.DOM 渲染在 哪个周期中就已经完成

mounted

6.简单描述每个周期具体适合哪些场景

生命周期钩子的一些使用方法：

  • beforecreate : 可以在这加个loading事件，在加载实例时触发
  • created : 初始化完成时的事件写在这里，如在这结束loading事件，异步请求也适宜在这里调用
  • mounted : 挂载元素，获取到DOM节点 updated : 如果对数据统一处理，在这里写上相应函数
  • beforeDestroy : 可以做一个确认停止事件的确认框

7.v-if和v-show的区别

v-if参数为false,整个标签被删除

v-show参数为false,标签样式被设为display:none

如果需要频繁切换状态的情况使用v-show,否则可以用v-if

8.为什么避免 v-if 和 v-for 用在一起

永远不要把 v-if 和 v-for 同时用在同一个元素上

当 Vue 处理指令时，v-for 比 v-if 具有更高的优先级

所以每次数据改变时,都会遍历for列表,浪费性能

解决方法:

  1. v-for的数据先在计算属性里处理一遍,只return需要渲染的数据
  2. v-if放在v-for的父盒子上面

9.指令v-el的作用是什么?

vue1.0中的v-el和v-ref在2.0中被废弃了。

提供一个在页面上已存在的 DOM 元素作为 Vue 实例的挂载目标.可以是 CSS 选择器，也可以是一个 HTMLElement 实例,

所以2.0中如果需要此功能要通过ref属性写法不是ref:而是ref=”名字”

用this.$refs获取元素

10.请说出vue.cli项目中src目录每个文件夹和文件的用法？

  • assets文件夹是放静态资源；
  • components是放组件；
  • router是定义路由相关的配置;
  • view视图；app.vue是一个应用主组件；
  • main.js是入口文件

Markdown

学习markdown时,练手抄的一篇教程.基本把语法过了一遍.

最后发现hexo对markdown语法支持不够简书友好,如标题,引用,分割线,图片等功能都有大大小小的bug.

文章转载自简书

Markdown是一种纯文本格式的标记语言。通过简单的标记语法，它可以使普通文本内容具有一定的格式。

相比WYSIWYG编辑器

优点:
  1. 因为是纯文本，所以只要支持Markdown的地方都能获得一样的编辑效果，可以让作者摆脱排版的困扰，专心写作。
  2. 操作简单。比如:WYSIWYG编辑时标记个标题，先选中内容，再点击导航栏的标题按钮，选择几级标题。要三个步骤。而Markdown只需要在标题内容前加#即可
缺点:
  1. 需要记一些语法（当然，是很简单。五分钟学会）。
  2. 有些平台不支持Markdown编辑模式。

还好，简书是支持Markdown编辑模式的。
