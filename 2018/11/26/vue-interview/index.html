3.请说下你对vue声明周期的理解?他的作用?

Vue 实例从创建到销毁的过程，就是生命周期。

作用是在vue实例创建和销毁时的几个阶段,调用对应的生命周期函数,可以在对应的时间里做一些操作,如计算数据、调用方法等.

它可以总共分为11个阶段：创建前/后, 载入前/后,更新前/后,keep-alive组件激活前/后,销毁前/销毁后,子孙组件报错时.

<keep-alive>是Vue的内置组件，能在组件切换过程中将状态保留在内存中，防止重复渲染DOM。

<keep-alive> 包裹动态组件时，会缓存不活动的组件实例，而不是销毁它们。

6.简单描述每个周期具体适合哪些场景

生命周期钩子的一些使用方法：

  • beforecreate : 可以在这加个loading事件，在加载实例时触发
  • created : 初始化完成时的事件写在这里，如在这结束loading事件，异步请求也适宜在这里调用
  • mounted : 挂载元素，获取到DOM节点 updated : 如果对数据统一处理，在这里写上相应函数
  • beforeDestroy : 可以做一个确认停止事件的确认框

7.v-if和v-show的区别

v-if参数为false,整个标签被删除

v-show参数为false,标签样式被设为display:none

如果需要频繁切换状态的情况使用v-show,否则可以用v-if

8.为什么避免 v-if 和 v-for 用在一起

永远不要把 v-if 和 v-for 同时用在同一个元素上

当 Vue 处理指令时，v-for 比 v-if 具有更高的优先级

所以每次数据改变时,都会遍历for列表,浪费性能

解决方法:

  1. v-for的数据先在计算属性里处理一遍,只return需要渲染的数据
  2. v-if放在v-for的父盒子上面

9.指令v-el的作用是什么?

vue1.0中的v-el和v-ref在2.0中被废弃了。

提供一个在页面上已存在的 DOM 元素作为 Vue 实例的挂载目标.可以是 CSS 选择器，也可以是一个 HTMLElement 实例,

所以2.0中如果需要此功能要通过ref属性写法不是ref:而是ref=”名字”

用this.$refs获取元素
